                                1. ERC20トークン

                                　下記のリンクからRemixを起動します。 https://remix.ethereum.org/?gist=c1d53cd3810b51016f9f03820f9210ca

                                xxxxxxxxxx
                                // SPDX-License-Identifier: MIT
                                pragma solidity ^0.8.9;
                                

                                import "@openzeppelin/contracts/token/ERC20/ERC20.sol";
                                import "@openzeppelin/contracts/access/Ownable.sol";
                                

                                contract PiyoPiyo is ERC20, Ownable {
                                    constructor() ERC20("PiyoPiyo", "PIYO") {}
                                

                                    function mint(address to, uint256 amount) public onlyOwner {
                                        _mint(to, amount);
                                    }
                                }

                                 

                                ソース解説

                                • ERC20トークンの作成に関してはOpenZeppelinのライブラリを用いるのが一般的です。

                                • コインの生成はコントラクト作成時に行うPremint方式と、コントラクトにmint関数を持たせるmintable方式があります

                                • その他のオプションについてはOpenZeppelinのドキュメントが参考になります

                                ERC20標準インターフェース

                                　今回のPiyoCoinでは上記に加えてmint関数を作成してある。この関数を使ってコントラクトオーナーが対象者に指定した残高のトークンを生成することができる。

                                 

                                PiyoCoinによるERC20操作

                                1. PiyoCoinの作成とデプロイ（これまでの手順で済んでいる）

                                2. コントラクトオーナー（リスト１番ユーザー）がリスト２番ユーザーに20000のコインをミントする

                                3. リスト２番ユーザーは自分の5000Piyoをリスト３番ユーザに転送する

                                4. リスト３番ユーザーは自分の1000Piyoをリスト４番ユーザに転送する許諾を与える

                                5. リスト４番ユーザーは自分がリスト３番ユーザーから引き出せる額が1000Piyoであることを確認する

                                6. リスト４番ユーザーはリスト３番ユーザーからリスト5番ユーザーへ500Piyoを第三者送金する

                                7. リスト４番ユーザーは自分がリスト３番ユーザーから引き出せる額が500Piyoであることを確認する

                                8. リスト５番ユーザーの残高を確認する

                                https://remix.ethereum.org/?gist=7c9bac1679e987798d30cd20b252a3d6

                                deploy済みの他のコントラクトを連携するサンプルです。

                                 

                                PiyoCoin.sol

                                xxxxxxxxxx
                                // SPDX-License-Identifier: MIT
                                pragma solidity ^0.8.0;
                                

                                import "@openzeppelin/contracts/token/ERC20/ERC20.sol";
                                

                                contract PiyoCoin is ERC20 {
                                

                                  address public minter;
                                

                                  constructor()  payable ERC20("PiyoCoin", "PIYO") {
                                    minter = msg.sender; //only initially
                                  }    
                                

                                  function passMinterRole(address _owner) public returns (bool) {
                                    require(msg.sender==minter, 'Error, only owner can change pass minter role');
                                    minter = _owner;
                                    return true;
                                  }
                                

                                  function mint(address payable _account, uint256 _amount) public payable {
                                    require(msg.sender==minter, 'Error, msg.sender does not have minter role'); //dBank
                                    _mint(_account, _amount);
                                  }
                                

                                }

                                 

                                PiyoHouse.sol

                                xxxxxxxxxx
                                // SPDX-License-Identifier: MIT
                                pragma solidity ^0.8.0;
                                

                                import "./PiyoCoin.sol";
                                

                                contract PiyoHouse {
                                

                                    // ピヨコインを内部トークンとして宣言します    
                                    PiyoCoin private pcoin;
                                    
                                    constructor(PiyoCoin _pcoin) {
                                        // マイグレート時にPiyoCoinコントラクトのアドレスを渡します
                                        pcoin = _pcoin;
                                    }
                                

                                    function ether2piyo() payable public {
                                        // レートは1ether = 20Piyo
                                        pcoin.mint(payable(msg.sender), msg.value * 10);
                                    }
                                }

                                2. PiyoBank

                                Piyoコインを使ったPiyoBankを作成します

                                 

                                仕様

                                • ethを預け入れることが可能です。

                                • す。

                                • 預け入れると期間に応じて、年利3%相当の利息が付与されます。

                                • 利息はPiyoCoinで支払われます。

                                • レートは固定で　1eth = 30piyo　です。

                                • 預け入れ中はethはロックされ、追加の預け入れはできません。

                                 

                                課題URL

                                https://remix.ethereum.org/?gist=0bd846114d932d564801fca1c9d423a9

                                 

                                ヒント

                                　PiyoBankではPiyoCoinのmint関数を呼び出すことで、利息の支払いを行います。このために、PiyoCoin.solをインポートして、内部で変数pcoinとしてインスタンスを生成しています。

                                 

                                3. PiyoLoan

                                　今度は、PiyoCoinをローンする機能をつけます。担保にはetherが必要です。

                                 

                                仕様

                                • Etherを担保にPiyoCoinを借りることができます。

                                • レートは1eth=30piyoです。

                                • 最低担保額は0.01ethです。

                                • ローンの返済時に返済金額の1%のetherが手数料として差し引かれます。

                                • ローン中は新たなローンを組むことができません。

                                 

                                課題URL

                                https://remix.ethereum.org/?gist=acb1860e79ea32617f30fb685f99eb37

                                 

                                 

                                4. 地震保険

                                　最後の課題は地震保険です。これは複数メンバでファンドを作り、地震が起きた際はそのファンドの最大30％が震度に応じて再配分される仕組みです。

                                仕様

                                • 加入者は地震保険に加入することができます。

                                • 一人につき1回のみ加入できます。

                                • 掛け金は最低0.01ethです。

                                • 加入時には市区町村コードを登録します。

                                • 地震が起きた際は市区町村コードと震度がペアとなった配列で関数がオーナーによって呼び出されます。

                                • 震度は0～9の10段階です（震度0,1,2,3,4,5弱,5強,6弱,6強,7）。

                                • 地震が起きた際は、まず全員の掛け金から最大震度に応じた割合でファンドを徴収します

                                • 割合は震度０が０％、震度７の時に30％となるように計算されます

                                • 徴収されたファンドは、対象地域の契約者に対して震度数に応じて分配されます。

                                • 震度数に応じて分配とは、支払い対象の契約についてその震度数をすべて足した総震度数を母数とした割合です。

                                • 契約者は契約を解除できますが、その時点での残高から1%が手数料として徴収されます。

                                • オーナーはすべての契約を強制的に解約（1%の手数料も取ります）し、すべてのethを取得することができます。

                                 

                                課題URL

                                https://remix.ethereum.org/?gist=05aa8c12996d613b9cc6f599404dd2cb

                                 

                                ヒント

                                • 契約内容は構造体を定義して配列で宣言するとよいでしょう。

                                • コントラクトのオーナーのアドレスをコントラクト生成時に保存しておきましょう。

                                • 地震時に呼ばれる関数と全契約解除はオーナーのみ実行できます。
